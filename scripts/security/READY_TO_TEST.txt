⚡ FASTEST ATTACKS (< 5 MINUTES):

  # Test 1: No authentication
  $ curl http://localhost:3000/api/users/profile

  # Test 2: Invalid token
  $ curl -H "Authorization: Bearer fake123" http://localhost:3000/api/users/profile

  # Test 3: Injection payload
  $ curl -X POST http://localhost:3000/api/bookings \
    -H "Content-Type: application/json" \
    -d '{"hotelName":"<img src=x onerror=alert(1)>"}'

  # Test 4: IDOR - try different user IDs
  $ for i in {1..10}; do
      curl -H "Authorization: Bearer $TOKEN" \
        http://localhost:3000/api/bookings/$i
    done

  # Test 5: Rate limiting
  $ for i in {1..100}; do
      curl -s http://localhost:3000/api/users/profile &
    done

✅ WHAT SECURE LOOKS LIKE:

  ✓ 401 without authentication header
  ✓ 401 with invalid token
  ✓ 400 with injection payloads (sanitized)
  ✓ 403 when accessing other users' data
  ✓ 429 after rate limit exceeded
  ✓ Security headers present
  ✓ No stack traces in error messages
  ✓ No server version/info leaked

❌ RED FLAGS (VULNERABILITIES):

  ✗ 200 status without auth (AUTH BYPASS!)
  ✗ Access other users' data (IDOR!)
  ✗ Injection payload echoed back (INJECTION!)
  ✗ Stack traces in errors (INFO DISCLOSURE!)
  ✗ 100 requests without 429 (NO RATE LIMIT!)
  ✗ Missing security headers
  ✗ Detailed error messages
  ✗ CORS allows * with credentials
